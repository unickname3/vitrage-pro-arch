# WordPress staging runbook (shared/VPS)

## 1. Подготовка окружения
- Создать поддомен staging, например `staging.vitrage-pro.ru`.
- Создать отдельную БД MySQL для staging.
- Включить HTTPS (Let's Encrypt).
- Ограничить доступ к staging через Basic Auth или allowlist IP.

## 2. Установка WordPress
1. Скачать актуальный WordPress в каталог staging.
2. Создать `wp-config.php` с отдельными staging-ключами и БД.
3. Задать префикс таблиц не `wp_` (например `vpstg_`).
4. Запретить индексацию:
   - В админке: Settings -> Reading -> discourage search engines.
   - Дополнительно в `robots.txt`: `Disallow: /`.

## 3. Минимальная безопасность
- Установить уникальные соли (`AUTH_KEY` и др.).
- Отключить редактирование файлов из админки:
  - `define('DISALLOW_FILE_EDIT', true);`
- Ограничить попытки входа (плагин или WAF хостинга).
- Включить 2FA для администраторов.
- Принудить сложные пароли.

## 4. Бэкапы
- Ежедневный backup БД + weekly backup `wp-content/uploads`.
- Хранение минимум 14 дней.
- Проверка восстановления 1 раз в месяц.

## 5. Обязательные плагины
- SEO: Rank Math или Yoast.
- Кеш: по рекомендациям хостинга.
- Медиа-оптимизация: WebP/сжатие.
- Формы: Contact Form 7 или Fluent Forms.
- ACF (для управляемых полей).

## 6. Релизный поток
1. Контент и верстка проверяются на staging.
2. Выполняется чек-лист QA/SEO.
3. Полный backup прода.
4. Перенос темы/настроек на прод.
5. Смоук-тест форм, страниц, редиректов и скорости.
